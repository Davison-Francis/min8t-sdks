What is Differential Loading?

Differential loading is a technique that serves optimized JavaScript bundles based on browser capabilities. Modern browsers receive smaller, faster ES2015+ code while legacy browsers get fully transpiled ES5 code with polyfills.

Modern Bundle (ES2015+)

Target: ~97% of users

Features: Arrow functions, classes, async/await, const/let

Size: ~25% smaller than ES5

Loaded via type="module"

Legacy Bundle (ES5)

Target: ~3% of users (IE11, old Safari)

Features: Full ES5 transpilation + polyfills

Size: Larger but fully compatible

Loaded via nomodule

How It Works: module/nomodule Pattern

Browsers automatically select the appropriate bundle based on their ES module support. No JavaScript user-agent detection needed!

<!-- Modern browsers load this (type="module") --> <script type="module" src="/dist/min8t.modern.js"></script> <!-- Legacy browsers load this (nomodule) --> <script nomodule src="/dist/min8t.legacy.js"></script> /* Modern browsers: - Execute type="module" script - Ignore nomodule script - Get 25% smaller bundle - Native ES2015+ features */ /* Legacy browsers (IE11, Safari < 11): - Ignore type="module" script (unsupported) - Execute nomodule script - Get full ES5 compatibility - All features work */

Technical Implementation

The min8t Plugin SDK uses Webpack multi-compiler mode with separate Babel configurations to generate both bundles:

Build Configuration

// webpack.config.differential.js module.exports = [ { /* Modern config */ output: { filename: 'min8t.modern.js' }, babel: { target: { esmodules: true } } }, { /* Legacy config */ output: { filename: 'min8t.legacy.js' }, babel: { target: { ie: 11 } } } ]; // Build command npm run build:differential

Babel Targets

Modern Babel Config

{ targets: { esmodules: true }, modules: false, useBuiltIns: false }

Legacy Babel Config

{ targets: { ie: 11 }, modules: false, useBuiltIns: false }
